# Feature: Core Command Infrastructure (Phase 8.1)

## Summary
Implement the foundational distributed command subsystem that provides unified command execution across all interfaces (CLI, TUI, web, IDE) using Horde for distributed process management and building upon existing registry patterns.

## Requirements
- [ ] Define CommandBehaviour with standardized callbacks for command implementation
- [ ] Create CommandMetadata structure for rich command description and parameters
- [ ] Implement CommandRegistry using Horde.Registry for distributed command registration
- [ ] Create CommandRouter for intelligent distributed command routing
- [ ] Build CommandHandler GenServer for distributed command execution
- [ ] Add command discovery and registration mechanisms
- [ ] Implement basic validation and error handling
- [ ] Integrate with existing InterfaceBehaviour and Interface.Gateway
- [ ] Maintain backward compatibility with existing Mix tasks
- [ ] Support interface-agnostic command definitions

### Technical Approach
1. **CommandBehaviour**: Define a behavior that all commands must implement
   - `metadata/0` - Returns command metadata (name, description, parameters)
   - `execute/2` - Executes command with params and context
   - `validate/1` - Validates command parameters before execution

2. **CommandMetadata**: Struct containing:
   - Command name, description, category
   - Parameter definitions with types and validation rules
   - Interface hints for UI generation
   - Examples and help text

3. **CommandRegistry**: Distributed registry using Horde.Registry
   - Register commands across the cluster
   - Support dynamic command discovery
   - Handle node join/leave events
   - Integrate with existing GlobalRegistry patterns

4. **CommandRouter**: Route commands to appropriate handlers
   - Use consistent hashing for command distribution
   - Support node affinity for stateful commands
   - Integrate with Interface.Gateway for request routing
   - Handle failover and retries

5. **CommandHandler**: GenServer for command execution
   - Supervised by Horde.DynamicSupervisor
   - Execute commands with timeout and monitoring
   - Support async and streaming responses
   - Integration with existing telemetry

## Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| Breaking existing Mix tasks | High | Implement alongside existing infrastructure, gradual migration |
| Command name conflicts | Medium | Namespace commands by category, validate uniqueness on registration |
| Distributed execution complexity | High | Start with local execution, add distribution incrementally |
| Interface compatibility | Medium | Design interface-agnostic from start, test with CLI adapter first |
| Performance overhead | Low | Use ETS caching for command metadata, benchmark routing performance |
