# Feature: Update mix.exs Files for Umbrella Applications

## Summary
Standardize and enhance mix.exs configurations across all umbrella applications to follow Elixir best practices, add missing metadata, and ensure consistent dependency management.

## Requirements
- [ ] Standardize Jason dependency version to ~> 1.4 across all apps
- [ ] Add missing inter-app dependencies (web needs engines and storage)
- [ ] Fix overly permissive postgrex version constraint (change from ">= 0.0.0" to "~> 0.17.0" or "~> 0.18.0")
- [ ] Add project metadata (description, package info, maintainers)
- [ ] Configure elixirc_paths for test environments
- [ ] Add standard aliases for common tasks (test, setup, format.check)
- [ ] Add development dependencies at root level for consistency
- [ ] Configure test coverage and documentation generation

## Research Summary
### Existing Usage Rules Checked
- No existing usage rules found for umbrella configuration

### Documentation Reviewed
- Elixir umbrella documentation: Dependencies and configuration are shared at root level
- Mix.Project documentation: Found complete list of available project options
- Hex package documentation: Package metadata requirements for publishing
- Phoenix/Hex examples: Real-world patterns for metadata and configuration

### Existing Patterns Found
- Pattern 1: [mix.exs:8-11] All apps use standard umbrella paths (../../_build, ../../deps, etc.)
- Pattern 2: [apps/*/mix.exs:21-23] All apps use OTP application with mod callback
- Pattern 3: Inter-app dependencies use `in_umbrella: true` pattern correctly

### Technical Approach
1. Update each app's mix.exs to:
   - Standardize dependency versions
   - Add missing inter-app dependencies  
   - Add elixirc_paths configuration for test environments
   - Add project metadata fields
   - Configure aliases for common tasks
   
2. Update root mix.exs to:
   - Add shared development dependencies (ex_doc, credo, dialyxir, excoveralls)
   - Configure test coverage settings
   - Add comprehensive aliases for umbrella-wide operations

3. Follow this update order:
   - Root mix.exs (shared deps and configs)
   - rubber_duck_core (base app)
   - rubber_duck_storage (depends on core)
   - rubber_duck_engines (depends on core)
   - rubber_duck_web (depends on all)

## Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| Breaking existing functionality | High | Run full test suite after each change |
| Version conflicts | Medium | Test dependency resolution with mix deps.get |
| Missing required fields | Low | Reference hex.pm docs for required metadata |
| Circular dependencies | Medium | Carefully plan inter-app dependencies |
