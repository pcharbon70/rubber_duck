# Feature: Git Hooks for Pre-Commit Checks

## Summary
Implement git pre-commit hooks that automatically run code quality checks before allowing commits, preventing commits with code formatting issues, linting errors, or compilation failures.

## Requirements
- [ ] Create pre-commit hook that runs existing quality checks
- [ ] Hook should run `mix check` (which includes format checking, credo, and compilation)
- [ ] Hook should exit with non-zero status to prevent commit if checks fail
- [ ] Hook should be fast enough for developer workflow (under 30 seconds)
- [ ] Hook should provide clear error messages when checks fail
- [ ] Hook should be easily installable for new developers
- [ ] Hook should handle both umbrella and individual app scenarios
- [ ] Hook should skip checks for merge commits and fixup commits when appropriate

### Technical Approach

#### Implementation Strategy
1. Create a pre-commit hook script that leverages existing `mix quality` alias
2. Use shell script for maximum compatibility and speed
3. Add proper error handling and user-friendly output
4. Include installation script for easy setup
5. Add configuration options for flexibility

#### Hook Structure
```bash
#!/bin/sh
# Run mix quality checks before commit
# Exit with non-zero if any check fails

echo "Running pre-commit quality checks..."

# Run the existing quality checks
if ! mix quality; then
    echo "❌ Pre-commit checks failed. Please fix the issues above before committing."
    echo "You can run 'mix quality' to see the same checks."
    echo "Or run 'mix format.all' to auto-fix formatting issues."
    exit 1
fi

echo "✅ All pre-commit checks passed!"
```

#### Installation Approach
- Create installation script that copies hook to `.git/hooks/pre-commit`
- Add mix alias for easy installation: `mix hooks.install`
- Add documentation for manual installation
- Make hook executable automatically

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Hook slows down commits | Medium | Use fast quality checks only, skip tests in pre-commit |
| Hook breaks developer workflow | High | Provide easy bypass option and clear error messages |
| Hook not installed by new developers | Medium | Add installation instructions to README and mix alias |
| Hook fails in CI/CD environments | Low | Design hook to work in both local and CI environments |
| Different behavior between hook and manual checks | Medium | Use exact same mix aliases in hook as developers use manually |
